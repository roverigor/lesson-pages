WSL no Windows

Ambiente Linux recomendado para AIOS

Usuários Mac/Linux podem pular esta seção. WSL (Windows Subsystem for Linux) é o ambiente recomendado para Windows. Ele garante compatibilidade total com as ferramentas do ecossistema AIOS.
1
Abra o PowerShell como Administrador
Clique com botão direito no menu Iniciar → Windows PowerShell (Admin)
2
Instale o WSL com Ubuntu
powershell 
wsl --install -d Ubuntu
3
Reinicie o computador
O Ubuntu abrirá automaticamente pedindo usuário e senha na primeira vez.
4
Instale Node.js no WSL
bash — dentro do terminal Ubuntu/WSL 
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verificar
node --version
npm --version
5
Configure permissões do npm
Evita erros de permissão ao instalar pacotes globais com npm install -g.
bash — dentro do terminal Ubuntu/WSL 
# Criar diretório para pacotes globais sem sudo
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# Verificar
npm --version
Performance no Windows: Mantenha seus projetos no diretório home do Linux (~/projetos/) e não em /mnt/c/.... O I/O é significativamente mais rápido.
3

Instalar Node.js

Necessário para AIOS Core, Codex CLI e Gemini CLI

Usuários Windows com WSL: o Node.js já foi instalado no passo 4 da seção anterior (dentro do Ubuntu). Podem avançar direto para a seção 4.
bash — Ubuntu / Debian / WSL 
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verificar
node --version   # deve mostrar v20.x.x ou superior
npm --version    # deve mostrar v9.x.x ou superior
bash — macOS via Homebrew 
brew install node@20

# Verificar
node --version   # deve mostrar v20.x.x ou superior
npm --version    # deve mostrar v9.x.x ou superior
powershell 
winget install OpenJS.NodeJS.LTS

# Verificar (abra um novo terminal após instalar)
node --version
npm --version
O Node.js instalado no Windows nativo não é acessível dentro do WSL. Se for usar WSL, instale o Node.js dentro do terminal Ubuntu (seção 2, passo 4).
Precisa gerenciar múltiplas versões do Node.js? Use o NVM (Linux/macOS) ou nvm-windows. Para a maioria dos usuários, a instalação direta acima é suficiente.

Instalar AIOS Core Principal

Um comando instala tudo — você só roda npx uma vez

npx é só na instalação. Você roda npx aios-core init apenas uma vez para criar o projeto. Depois disso, o AIOS já está instalado localmente e você usa os agentes direto na IDE (Claude Code, Cursor, etc.) sem precisar rodar npx novamente. Pense no npx como o instalador de um programa — você usa uma vez e depois abre o programa normalmente.
bash 
# Criar novo projeto com assistente interativo
npx aios-core init meu-projeto

# Ou com versão específica
npx aios-core@latest init meu-projeto
bash 
# Instalar em projeto existente
cd seu-projeto
npx aios-core install

# O instalador detecta automaticamente sua configuração
bash 
# Upstream Auto-Update — atualiza sem perder customizações
npx aios-core@latest install

# Simular antes (ver o que muda sem aplicar nada)
npx aios-core install --dry-run

# Verificar saúde após atualizar
npx aios-core doctor

# Corrigir problemas automaticamente (se houver)
npx aios-core doctor --fix
O que o updater faz automaticamente: detecta a instalação existente, faz backup das suas customizações em .aios-backup/, atualiza apenas os arquivos do core que mudaram e preserva suas configurações, squads e templates personalizados.

O que o instalador faz

Download da versão mais recente do NPM
Assistente interativo com @clack/prompts (escolha de IDE, gerenciador de pacotes)
Configuração automática do IDE (Codex CLI, Cursor ou Claude Code)
Setup de todos os agentes e workflows AIOS no projeto
Criação dos arquivos de configuração necessários
Inicialização do sistema de meta-agentes
Verificações de saúde do sistema
Suporte a Windows, macOS e Linux

Estrutura instalada

estrutura 
meu-projeto/
├── .aios-core/          # Framework AIOS (não modificar)
│   ├── agents/              # Agentes: @dev, @qa, @architect, @pm...
│   ├── tasks/               # Tasks executáveis
│   ├── workflows/           # Workflows multi-step
│   └── templates/           # Templates de docs
├── docs/
│   └── stories/             # Stories de desenvolvimento
└── expansion-packs/     # Suas extensões customizadas

AIOS Dashboard Opcional

Interface web de observabilidade — precisa de AIOS Core instalado

Em desenvolvimento inicial (Early Stage). O Dashboard é funcional mas pode ter mudanças. Contribuições são bem-vindas!
O AIOS funciona 100% sem o Dashboard. Ele existe apenas para observar o que acontece na CLI — nunca para controlar. A CLI é sempre a fonte da verdade.

Instalar o Dashboard

bash 
# Dentro do projeto com AIOS instalado
mkdir -p apps
git clone https://github.com/SynkraAI/aios-dashboard.git apps/dashboard

# Dependências do Dashboard (Next.js)
npm install --prefix apps/dashboard

# Dependências do Server (requer Bun — https://bun.sh)
cd apps/dashboard/server
bun install

Iniciar (2 terminais)

bash — Terminal 1: Server de eventos (porta 4001) 
cd apps/dashboard/server
bun run dev
# Saída esperada: 🚀 Monitor Server running on http://localhost:4001
bash — Terminal 2: UI do Dashboard (porta 3000) 
npm run dev --prefix apps/dashboard
# Saída esperada: ▲ Next.js — Local: http://localhost:3000
Requer Bun. O server do Dashboard usa Bun como runtime. Instale em bun.sh antes de rodar o server. O Dashboard em si (Next.js) usa npm normalmente.
Abra http://localhost:3000 no navegador.

CodeRabbit AI Destaque

Code review com IA — integrado ao fluxo do agente @qa

CodeRabbit é o MCP de code review do fluxo AIOS. Ele é usado pelo agente @qa e pelo ciclo de desenvolvimento (@dev → PR → CodeRabbit → @qa) para revisar Pull Requests automaticamente com contexto do código e da story em andamento.
Onde é usado no AIOS
Revisão automática de PRs criados pelo @dev
Agente @qa lê comentários do CodeRabbit para validação
Checklist de Definition of Done (DoD) usa relatório CodeRabbit
@sm incorpora feedbacks no refinamento de stories
Reduz ciclos manuais de review — o time foca em decisões, não em achados
Por que é importante
Analisa cada PR com contexto semântico do diff
Detecta bugs, code smells e violações de padrão
Gera comentários line-by-line acionáveis
Integra com GitHub/GitLab sem configuração extra
Gratuito para repositórios open source
bash — configurar CodeRabbit MCP 
# Adicionar CodeRabbit MCP ao Claude Code
# Em ~/.claude.json, adicione em mcpServers:
"coderabbit": {
  "type": "stdio",
  "command": "npx",
  "args": ["-y", "@coderabbitai/mcp"],
  "env": {
    "CODERABBIT_API_KEY": "sua-chave-aqui"
  }
}

# Obter chave API em: https://coderabbit.ai
# Gratuito para repositórios open source
Após configurar o MCP, o agente @qa pode chamar *review-pr e obter o relatório completo do CodeRabbit diretamente no fluxo AIOS, sem precisar abrir o GitHub manualmente.

Troubleshooting

Problemas comuns e soluções

npx aios-core não encontrado
Verifique se Node.js ≥ 18 está instalado: node --version
Agente não responde ao @agent-name
Confirme que o AIOS foi instalado no projeto atual. O .aios-core/ deve existir na raiz do projeto onde você abriu o Claude Code.
Erro de permissão no npm install global
bash 
# Configure npm para instalar sem sudo
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
Claude Code não reconhece AIOS após instalação
Feche e reabra o Claude Code na pasta do projeto. O AIOS é carregado quando o CLI é iniciado no diretório correto.
Preciso rodar npx toda vez que abrir o projeto?
Não. O npx é usado apenas para instalar ou atualizar o AIOS. No dia a dia, você abre o CLI (claude, codex ou gemini) dentro do projeto e usa os agentes diretamente com @agent-name. O framework já está instalado localmente na pasta .aios-core/.
Dashboard não inicia / erro no server
O server do Dashboard requer Bun como runtime. Verifique:
bash 
# Verificar se Bun está instalado
bun --version

# Se não estiver, instale:
curl -fsSL https://bun.sh/install | bash

# Reinicie o terminal e tente novamente
cd apps/dashboard/server
bun install
bun run dev
Problemas não listados aqui? Abra uma issue em github.com/SynkraAI/aios-core/issues
AIOS instalado com sucesso?
Ative o @aios-master e rode *help para ver tudo que está disponível.
