Glossário Técnico

57 termos técnicos extraídos das Aulas 02 e 03, organizados por categoria.

Ferramentas

Squad Creator Aula 02

Sistema CLI para criar, configurar e gerenciar times de agentes AI que trabalham juntos para executar tarefas complexas. Versão 3.0-4.0 com dashboard interativo e otimização de custos.

Squad Creator Pro Aula 03

Versão avançada do Squad Creator com dashboard visual de squads, visualização de workflows como diagramas, monitor de agentes em tempo real, Finch (análise de negócios) e sistema de tiers. Versão 4.0 com ~67 comandos.

Domain Decoder Aula 03

Ferramenta ou módulo do AIOS para decodificar e mapear domínios de conhecimento. Relacionado à capacidade de extrair estruturas de conhecimento de um domínio específico.

Finch Aula 03

Módulo do Squad Creator Pro que faz análise de negócios sobre squads: quanto pode economizar, quanto pode faturar, viabilidade financeira.

ffmpeg Aula 03

Ferramenta de linha de comando para processamento de áudio e vídeo. Usada para edição de vídeo via código dentro de squads de produção de conteúdo.

Modelos LLM

Haiku Aula 02

Claude Haiku — modelo mais rápido e barato da Anthropic. Input: $0.25/1M tokens, Output: $1.25/1M tokens. Surpreendentemente capaz em 70-80% das tarefas quando bem orientado.

Sonnet Aula 02

Claude Sonnet — modelo intermediário. Balanceado entre custo e capacidade. Input: $3/1M tokens, Output: $15/1M tokens. 15-20% dos casos.

Opus Aula 02

Claude Opus — modelo mais poderoso. Input: $15/1M tokens, Output: $75/1M tokens. Para raciocínio complexo, multi-step, apenas 5-10% dos casos.

Conceitos

Agent Aula 02

Entidade AI com persona, expertise e ferramentas específicas. Ativado com @agent-name (ex: @dev, @qa).

Token Aula 02

Unidade de cobrança dos modelos LLM. ~4 caracteres = 1 token em inglês. Input e output tokens têm preços diferentes.

Operação Determinística Aula 02

Tarefa que pode ser resolvida algoritmicamente sem LLM. Ex: validar JSON, contar palavras, parsing com regex. Custo zero vs $0.01-0.05 por LLM call.

Workflow Aula 02

Sequência de tarefas executadas por agentes em ordem específica. Pode ter paralelização, condicionais, loops. Definido em YAML ou código.

Epic Aula 02

Grande feature dividida em múltiplas stories. No AIOS, pode ser executado automaticamente por squads.

Story Aula 02

Unidade de trabalho implementável em 1-3 dias. Contém contexto, critérios de aceitação, tasks e testes.

Context Window Aula 02

Número máximo de tokens que um modelo processa de uma vez. Claude: 200k tokens (~150k palavras).

Persona Aula 02

Identidade e comportamento definido para um agente: expertise, tom de voz, valores, restrições.

Voice DNA Aula 02

Assinatura linguística única de um agente: frases preferidas, metáforas recorrentes, estrutura de pensamento.

System Prompt Aula 02

Instrução inicial que define comportamento, persona e regras de um agente AI. Permanece ativo durante toda a sessão.

Idempotent Operation Aula 02

Operação que produz mesmo resultado se executada 1x ou N vezes. Importante para retry logic e confiabilidade.

Observabilidade Aula 03

Capacidade de monitorar o que agentes estão fazendo em tempo real. Segundo princípio do AIOS (após CLI First). Via dashboard, terminal ou skill /squad.

Advisory Board Aula 03

Conselho consultivo formado por clones de mentes (minds) que debatem entre si para tomada de decisão estratégica.

Spawnar Aula 03

Criar e ativar um subagente ou subprocesso a partir de um agente principal. O contexto atual é copiado (fork) para o novo processo.

Worker Online Aula 03

Processo ou serviço que roda em segundo plano num servidor remoto (ex: Vercel, Railway), executando tarefas de forma contínua 24/7.

Design System Aula 03

Sistema visual padronizado para squads e aplicações. Inclui componentes reutilizáveis, padrões visuais e hierarquia de elementos.

Repertório Aula 03

Estoque de referências e experiências que determina a capacidade de avaliar qualidade dos outputs. Criar squads na área de seu repertório gera resultados muito superiores.

Abstração Aula 03

Capacidade de simplificar complexidade, escondendo detalhes e expondo apenas o necessário. Camadas de abstração em UI limitam poder da IA.

Componentização Aula 03

Quebrar sistemas complexos em componentes menores, reutilizáveis e independentes. Aplica-se a squads, código e serviços.

Front Matter Aula 03

Metadados no topo de arquivos Markdown (entre --- e ---) que configuram comportamento de agentes e tasks. Define modelo a usar, permissões, registros de memória.

Kanban (contexto AIOS) Aula 03

Visualização do progresso no dashboard AIOS. Colunas: Backlog, Em Progresso, Review, Humano Necessário, PR Criado, Done.

Estratégias

Token Economy Aula 02

Conjunto de técnicas para reduzir custos de LLM: modelos baratos quando possível, scripts determinísticos, batch processing, caching. Redução de 70-90%.

Fallback Aula 02

Mecanismo automático de contingência. Se provider A falhar, o sistema automaticamente tenta provider B, depois C, etc.

Prompt Engineering Aula 02

Arte de construir prompts eficazes para LLMs. Boa engenharia permite usar Haiku em vez de Opus. Pode melhorar qualidade em 50-200% e reduzir custo em 90%.

Técnicas

Batch Processing Aula 02

Processar múltiplos itens de uma vez. Reduz overhead de API calls e melhora custo-benefício.

Chunking Aula 02

Dividir documentos grandes em pedaços menores para processar. Essencial quando conteúdo excede context window.

Caching Aula 02

Armazenar resultados de operações caras para reusar. 90-99% economia em cenários de uso repetitivo.

Streaming Aula 02

Receber resposta do LLM incrementalmente (palavra por palavra). Melhora perceived performance e permite early stopping.

Comandos

Squad Validate Aula 02

Testa configuração e funcionamento de um squad com exemplos pequenos antes de escalar.

Squad Optimize Aula 02

Analisa uso de LLM e sugere otimizações: downgrade de modelos, conversão para scripts, caching.

*create-squad Aula 03

Criar novo squad do zero. Na v4.0, define automaticamente os três tiers, verifica registry e sugere ferramentas via discovery-tools.

*validate-squad Aula 03

Valida qualidade e estrutura de um squad. Gera nota de 0-10 com relatório detalhado. Nota > 8.5 = pronto para produção.

Discovery Tools Aula 03

Pesquisa na Internet e descobre APIs, MCPs, projetos GitHub relevantes para um squad ou agente.

*sync commands Aula 03

Sincroniza os comandos do Squad Creator no ambiente .claude/commands/. Necessário após atualizar versão.

Arquitetura

Agents & Skills Aula 03

Nova arquitetura que substitui Commands e Prompts. Agents com @ (ex: @dev), Skills com / (ex: /squad). Agentes se chamam automaticamente entre si.

Squad Tiers Aula 03

Classificação hierárquica de squads: Estratégico (C-levels, visão), Tático (gestores, processos), Operacional (execução, entrega).

Filosofia

CLI First Aula 03

Priorizar linha de comando sobre interface gráfica. Quanto mais UI, menos poder a IA tem. IA trabalha melhor com Markdown, YAML e JSON.

UI First Aula 03

Abordagem oposta ao CLI First. Considerada limitante no AIOS porque camadas visuais reduzem poder da IA. Cursor (UI) vs Claude Code (CLI).

Serviços & Infraestrutura

Open Router Aula 02

API unificada com acesso a 20+ provedores de LLM. Uma API key, múltiplos modelos, fallback automático entre provedores.

Dashboard Aula 02

Interface web interativa: agentes, workflows, métricas de execução, custos em tempo real e logs.

ETL Pipeline Aula 02

Extract Transform Load — padrão de processamento de dados. Extrair de fonte, transformar/enriquecer, carregar em destino.

MCP (Model Context Protocol) Aula 02

Protocolo para conectar LLMs a ferramentas externas (APIs, bancos de dados, file system). Permite agentes interagirem com o mundo real.

LLM Call Aula 02

Requisição a um modelo de linguagem. Tem custo por token e latência. Deve ser usado apenas quando raciocínio é necessário.

Registry Aula 03

Documento central que lista todos os squads existentes, suas ferramentas e capacidades. Evita duplicatas ao criar novos squads.

Sistemas, Métricas & Parâmetros

Model Tier Qualification Aula 02

Sistema automático que classifica tarefas em Tier 1 (Haiku), Tier 2 (Sonnet) ou Tier 3 (Opus) baseado em complexidade.

Rate Limiting Aula 02

Limite de requisições por minuto/hora imposto pelos provedores de API. Open Router com fallback resolve elegantemente.

ROI Aula 02

Retorno sobre investimento. Se gasta $10/mês, não vale otimizar. Se gasta $10k/mês, vale muito.

Throughput / Latency Aula 02

Throughput: volume de trabalho por unidade de tempo. Latency: tempo entre requisição e resposta. Haiku ~1-3s, Sonnet ~3-8s, Opus ~8-20s.

Temperature / Top P / Max Tokens Aula 02

Parâmetros de LLM. Temperature: 0 = determinístico, 1+ = criativo. Top P: considera top N% de tokens mais prováveis. Max Tokens: limite de resposta.

Bob (agente) Aula 03

Agente autônomo avançado que executa múltiplos passos sozinho, orquestrando outros agentes automaticamente. Não usar para criação de squads. Previsto para a décima aula.

FAQ

30 perguntas frequentes extraídas das Aulas 02 e 03, organizadas por categoria.

Instalação e Configuração

Como instalo o Squad Creator?
Instale via npm globalmente: npm install -g @aios/squad-creator. Depois configure API keys (Anthropic + Open Router recomendado) e verifique com squad --version.
Preciso de todas as API keys ou só Anthropic basta?
Mínimo: Apenas Anthropic. Recomendado: Anthropic + Open Router. Open Router dá acesso a 20+ provedores com fallback automático e billing unificado.
O dashboard não abre automaticamente, como acesso?
Use squad dashboard ou acesse http://localhost:3000. Em WSL2, use IP do WSL.

Atualização do Squad Creator 8+ alunos perguntaram

Como atualizo o Squad Creator para a versão mais recente?
1) Baixe a nova versão do WhatsApp. 2) Delete APENAS a pasta /squads/squad-creator/ (NÃO outros squads!). 3) Extraia a nova versão. 4) Sincronize: *sync commands squad-creator. 5) Verifique. Erro comum (Fábio): Deletou toda a pasta /squads/ e teve que restaurar via git.
Como uso o comando *sync commands squad-creator?
Lê os arquivos do /squads/squad-creator/ e cria/atualiza os comandos em .claude/commands/. Use após instalar nova versão ou quando comandos não aparecem com /. Alternativa: chamar @devops.

Token Economy e Custos

Por que Haiku às vezes dá resultado melhor que Opus?
Haiku é mais focado e direto. Opus tende a adicionar complexidade desnecessária em tarefas simples. Com prompt engineering otimizado para Haiku, resultados são excelentes em 70-80% dos casos.
Quanto posso economizar realmente usando essas técnicas?
Cenário real: 500 transcrições com Opus: $720. Com Haiku + scripts: $12 (economia de 98.3%). Resultado típico: 70-90% de redução mantendo ou melhorando qualidade.
Vale a pena otimizar se meu gasto mensal é pequeno?
< $10/mês: não vale. $10-50: otimizações simples. $50-500: vale MUITO, ROI em 1-4 semanas. > $500: ESSENCIAL.
O Squad Creator Pro consome muitos tokens ao abrir?
Marcelo reportou 106k tokens; Wanderson teve 35k. Agents & Skills fazem fork e releem contexto. Use /context para verificar. Pedro está otimizando via Front Matter e scripts em background.

Estrutura de Pastas e Organização

A pasta dos squads fica dentro do .claude ou fora?
Fora do .claude, numa pasta /squads/ na raiz do projeto. Os comandos são sincronizados para .claude/commands/ via *sync commands. José Carlos: “O / você ativa mudando no config.json.”
Como organizar squads em múltiplos projetos?
Opções: 1) Copiar squads para cada projeto (simples). 2) Pasta raiz compartilhada. 3) Skill de sync do GitHub (Torriani criou). Pedro recomenda AIOS na pasta de cada projeto para contexto isolado.

Model Tier & Operações

Como o sistema decide qual modelo usar para cada tarefa?
Model Tier Qualification classifica por complexidade, histórico, análise do prompt e output esperado. Override manual sempre permitido: squad run --model haiku.
Como sei se uma operação deve ser script ou LLM call?
Regra de ouro: Se consegue escrever if/else ou regex, deve ser script. Scripts = custo zero. LLM = quando precisa de raciocínio semântico, classificação, resumo, geração de texto.

Entrega e Deploy de Squads

Como fazer deploy/entrega de squad para cliente?
Nível 1 — Consultoria: Entregar squad desenhado + .md de cada agente (Edu já validou). Nível 2 — SaaS: Rodar o squad no seu ambiente e entregar outputs. Nível 3 — App completo: Criar interface com banco de dados, APIs e deploy em servidor.

Conceitos de Squad

Squad é a mesma coisa que um setor da empresa?
Pode ser, mas não é só isso. Squad pode ser setor, grupo de elite para missão específica, ou time multidisciplinar autônomo. Com Tiers: Estratégico (decisões), Tático (processos) e Operacional (execução).
Por que é melhor começar com Squad do que com aplicação web?
Analogia Agile do Pedro (skate → carro): Squad valida task por task sem latência de desenvolvimento. Sem complexidade de APIs/banco. Depois só adiciona ferramentas. “Código é commodity. Framework validado que gera valor é o que importa.”
Poucos agentes com workflows ou muitos agentes sem workflows?
Workflows definidos > quantidade de agentes. Squad = agentes + tasks + workflows + checklists para cumprir um propósito. Exemplo real: DicOp tem 24 agentes, 59 tasks, 14 workflows, 32 checklists.

CLI First e Agents & Skills

O que muda com a migração de Commands para Agents & Skills?
Agents com @ (ex: @dev), Skills com / (ex: /squad). Agentes se chamam automaticamente entre si. Mais poder e velocidade. NÃO migre por conta própria — aguarde atualização oficial.
Por que CLI First e não UI First?
IA trabalha melhor com Markdown, YAML, JSON. Menos camadas de abstração = mais poder. Cursor (UI) está perdendo para Claude Code (CLI). Dashboard é READ-ONLY para observabilidade.

Repertório e Criação de Squads

Qual squad devo criar primeiro?
Regra de ouro: “Crie o squad onde você tem REPERTÓRIO para avaliar se ele é bom.” IA é muito boa em fingir que as coisas estão certas. Sem repertório, você não sabe se o output é bom. Comece pelo que você domina.

Validação, Performance e Troubleshooting

Por que validar com 10-20 exemplos antes de escalar?
Evita: prompt mal configurado ($1 vs $100+), modelo errado (economia de 98%+), workflow com bug (30min vs 2 dias). 30 minutos de validação economizam dias de retrabalho.
Meu squad está lento, como acelero?
Gargalos comuns: modelo pesado demais (use Model Tier), sequencial demais (paralelizar), rate limiting (Open Router), LLM calls desnecessárias (converter em scripts), context window grande (chunking), sem caching.
Meu squad falha aleatoriamente, como debugo?
1) Debug mode: squad run --debug. 2) Logs: squad logs --filter error. Falhas comuns: Rate Limiting (429), Timeout, API Key inválida, Context Too Long.
Open Router custa mais caro que APIs diretas?
Não, geralmente mais barato ou igual. Sem markup na maioria dos modelos. Vantagens extras: uma API key = 20+ providers, fallback automático, billing unificado. Desvantagem: ~50-100ms de latência adicional.
