# Story 4.3 — Admin UI: tab Avaliações + surveys.js

**Epic:** EPIC-004  
**Status:** Ready  
**Type:** Frontend / Admin  
**Depende de:** Story 4.1, 4.2

## O que fazer

Adicionar tab "Avaliações" ao painel admin com módulo `surveys.js` que gerencia criação, disparo e visualização de resultados de surveys NPS/CSAT.

## Acceptance Criteria

### Tab e estrutura
- [ ] Nova tab "Avaliações" no `admin.html` (ícone: 📊 ou similar)
- [ ] Módulo `js/admin/surveys.js` carregado no admin.html
- [ ] Tab renderiza: botão "+ Nova Avaliação" + lista de surveys

### Criação de Survey (modal)
- [ ] Campos: Nome da avaliação, Tipo (NPS 0-10 / CSAT 1-5 estrelas), Turma (dropdown de classes), Pergunta principal (textarea), Follow-up aberto (textarea, opcional)
- [ ] Ao salvar: INSERT em `surveys` com status='draft'
- [ ] Survey nova aparece na lista com status "Rascunho"

### Lista de Surveys
- [ ] Cada survey exibe: nome, tipo badge (NPS/CSAT), turma, status, data de disparo (se disparado), contador "X/Y responderam"
- [ ] Botão "Disparar" (só para status=draft ou active): chama `dispatch-survey` Edge Function
- [ ] Ao disparar: confirma com modal ("Enviar para N alunos via WhatsApp?") → executa → exibe resultado (N enviados, M sem phone)
- [ ] Botão "Ver Resultados" → abre drawer/painel lateral
- [ ] Botão "Encerrar" (status active → closed)

### Drawer de Resultados
- [ ] Para NPS: exibe NPS Score calculado ((promotores% - detratores%) * 100), distribuição em barras (detratores 0-6, neutros 7-8, promotores 9-10)
- [ ] Para CSAT: exibe média, distribuição de notas 1-5 com barras
- [ ] Lista de comentários (feedback aberto) abaixo
- [ ] Total responderam / total enviado

### Exportação de Respostas
- [ ] Botão "Exportar CSV" no drawer de resultados: baixa arquivo `avaliacao-{nome}-{data}.csv`
  - Colunas: data_resposta, nome_aluno, telefone, turma, tipo, nota, comentario
- [ ] Botão "Exportar MD" no drawer de resultados: baixa arquivo `avaliacao-{nome}-{data}.md`
  - Header com nome da survey, turma, score geral e data de exportação
  - Tabela markdown com as respostas
  - Seção de comentários listados abaixo da tabela
- [ ] Exportação acontece no browser (sem chamada ao servidor): dados já carregados no drawer → `Blob` + `URL.createObjectURL` + click em `<a download>`
- [ ] Botão "Exportar" também disponível na lista de surveys (sem precisar abrir drawer)
