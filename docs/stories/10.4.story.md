## Descrição

Como **aluno da Academia Lendária**, quero receber uma mensagem no WhatsApp quando a gravação da minha aula fica disponível — sem precisar verificar o painel manualmente — para que eu possa assistir a aula que perdi ou revisar o conteúdo na hora certa.

**Contexto técnico:** O webhook `recording.completed` (Story 10.1) salva a gravação em `class_recordings`. Ao salvar, um trigger envia WhatsApp via Evolution API para todos os alunos do cohort associado. Reutiliza o padrão de anti-spam da Story 6.4 — sem envios duplicados para o mesmo `recording_id`.

---

## Acceptance Criteria

- [x] **AC1 — Notificação disparada:** quando um registro é criado em `class_recordings` com `cohort_id` válido, todos os alunos do cohort com `whatsapp_alerts_enabled = true` recebem mensagem
- [x] **AC2 — Mensagem personalizada:** inclui nome do aluno, nome da turma, título da aula e link da gravação
- [x] **AC3 — Anti-spam:** cada gravação gera no máximo 1 mensagem por aluno (controle por `recording_id` em tabela de log)
- [x] **AC4 — Respeita opt-out:** `students.whatsapp_alerts_enabled = false` → não envia
- [x] **AC5 — Número válido:** verifica `students.phone` não nulo antes de enviar
- [x] **AC6 — NUNCA enviar para números reais em teste** — usar SEMPRE `554399250490` (Igor) durante dev/debug
