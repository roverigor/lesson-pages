# Story 4.1 — DB Schema: surveys + survey_links + RLS

**Epic:** EPIC-004  
**Status:** Ready  
**Type:** Database

## O que fazer

Criar as tabelas `surveys` e `survey_links`, adicionar colunas em `student_nps`, e configurar RLS para o fluxo público de resposta.

## Acceptance Criteria

- [ ] Tabela `surveys` criada com colunas: id, name, type (nps/csat), cohort_id, class_id, question, follow_up, status, dispatched_at, created_by, created_at
- [ ] Tabela `survey_links` criada com colunas: id, survey_id, student_id, token (UUID único), used_at, created_at + UNIQUE(survey_id, student_id)
- [ ] `student_nps` recebe colunas `survey_id` e `survey_type`
- [ ] RLS em `surveys`: admin lê e escreve (authenticated + role=admin)
- [ ] RLS em `survey_links`: admin lê/escreve + anônimo pode SELECT por token (para validar)
- [ ] RLS em `student_nps`: INSERT público permitido (anônimo) — submissão via Edge Function com service role
- [ ] Índices em survey_links(token), survey_links(survey_id), surveys(cohort_id), surveys(status)
- [ ] Migration aplicada via `supabase db push`
