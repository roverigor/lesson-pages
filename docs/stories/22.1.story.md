## Story
Como **administrador da Academia Lendária**,
Eu preciso que **todas as 3 tabelas que representam aluno (`students`, `student_imports`, `wa_group_members`) tenham coluna `normalized_phone` em formato E.164 (+55DDD9XXXXXXXX) garantida por trigger + VIEW canônica `v_students_unified` que mescle as 3 fontes**,
Para que **campanhas (NPS, lembretes, PS RSVP) usem 1 identidade única por aluno e nenhum aluno seja perdido por phone format divergente**.

## Contexto

Discovery `02-db-audit.md` §3.1 identificou **triple-identity** crítico:

| Tabela | Origem | Phone key |
|--------|--------|-----------|
| `students` | CSV manual + admin entries | UNIQUE(phone, cohort_id) — formato heterogêneo |
| `student_imports` | AC webhook payload JSONB | Sem constraint — pode ter "+55", "0x", espaços |
| `wa_group_members` | Sync cron WA group | UNIQUE(cohort_id, phone) — format WA API |

**Problema raiz:** phone formats divergem (`+5511987654321` vs `11987654321` vs `5511987654321`). JOINs por phone falham silenciosamente. `dispatch_link_opens` + `nps_class_links` não sabem qual `student_id` usar. Campanhas perdem alunos (impacto direto: receita NPS + retenção).

**Estado atual:**
- Migração `20260409050000_normalize_student_phones.sql` definiu `normalize_student_phones()` RPC — mas SEM trigger automático
- Backfill incompleto: novos alunos via AC ou WA podem entrar com phone não-normalizado
- Não existe VIEW canônica — clientes (edge functions, RPCs) precisam JOIN manual com lógica frágil

**Decisão arquitetural confirmada:**
- `students` = **source-of-truth** (admin canonical)
- `student_imports` = **audit log AC** (nunca deletar — auditoria de compras)
- `wa_group_members` = **espelho WA** (sync automático)
- VIEW `v_students_unified` resolve identidade por `(normalized_phone, cohort_id)`

**Sistema EM PRODUÇÃO** com 1k+ alunos ativos. Backfill exige idempotência + dry-run + rollback + smoke + Slack alert + janela manutenção (low-traffic).

## Acceptance Criteria

### AC1 — Coluna `normalized_phone` em 3 tabelas
**Given** schema atual (sem coluna normalizada explícita)
**When** migration aplica
**Then** `students`, `student_imports`, `wa_group_members` têm coluna `normalized_phone text` (nullable durante backfill, NOT NULL após T6)

### AC2 — Função `normalize_phone_e164(text) returns text`
**Given** input phone em qualquer format ("+5511987654321", "11 98765-4321", "5511987654321", "+55 (11) 98765-4321")
**When** chama `normalize_phone_e164(input)`
**Then** retorna E.164 `+55DDD9XXXXXXXX` (13 dígitos pós-`+`)
**E** retorna NULL se input inválido (< 10 dígitos ou > 14 dígitos após strip)

### AC3 — Trigger BEFORE INSERT/UPDATE
**Given** trigger criado nas 3 tabelas
**When** INSERT ou UPDATE com `phone` setado
**Then** `normalized_phone` é populado automaticamente via `normalize_phone_e164(NEW.phone)`
**E** trigger não falha se `phone` é NULL (deixa `normalized_phone` NULL também)

### AC4 — Backfill idempotente
**Given** dados existentes nas 3 tabelas
**When** roda RPC `backfill_normalized_phones()` 1x ou 2x
**Then** rows com `normalized_phone IS NULL` recebem valor populado
**E** rows com `normalized_phone` já populado NÃO são modificadas (idempotente)
**E** log estruturado mostra count atualizado por tabela

### AC5 — VIEW `v_students_unified`
**Given** dados normalizados nas 3 tabelas
**When** query `SELECT * FROM v_students_unified WHERE cohort_id = 'T5'`
**Then** retorna 1 linha por `(normalized_phone, cohort_id)` mesclando:
- `students` columns (canonical id, name, is_mentor, active)
- `student_imports` columns (product, source, ac_event_id)
- `wa_group_members` columns (wa_name, group_id, synced_at)
**E** colunas conflitantes (ex: `name`) priorizam `students.name` > `wa_group_members.wa_name` > `student_imports.name`

### AC6 — UNIQUE(normalized_phone, cohort_id) em `students`
**Given** backfill completo + sem duplicates
**When** migration adiciona constraint
**Then** `ALTER TABLE students ADD CONSTRAINT students_normalized_phone_cohort_unique UNIQUE (normalized_phone, cohort_id)` passa
**E** rodar `find_duplicate_students()` RPC retorna 0 duplicates

### Templates SQL

#### Função normalize_phone_e164
```sql
CREATE OR REPLACE FUNCTION public.normalize_phone_e164(input text)
RETURNS text
LANGUAGE plpgsql
IMMUTABLE
AS $$
DECLARE
  cleaned text;
  digits_only text;
BEGIN
  IF input IS NULL OR length(trim(input)) = 0 THEN
    RETURN NULL;
  END IF;

  -- Strip everything except digits and leading +
  cleaned := regexp_replace(input, '[^\d+]', '', 'g');
  digits_only := regexp_replace(cleaned, '^\+', '', 'g');

  -- Length-based normalization
  IF length(digits_only) < 10 OR length(digits_only) > 13 THEN
    RETURN NULL;
  END IF;

  -- Already starts with 55 (BR country code)?
  IF substring(digits_only, 1, 2) = '55' AND length(digits_only) IN (12, 13) THEN
    cleaned := '+' || digits_only;
  ELSIF length(digits_only) IN (10, 11) THEN
    cleaned := '+55' || digits_only;
  ELSE
    RETURN NULL;
  END IF;

  -- Final pattern validation E.164 BR
  IF cleaned ~ '^\+55[1-9][1-9][0-9]{8,9}$' THEN
    RETURN cleaned;
  ELSE
    RETURN NULL;
  END IF;
END;
$$;
```

#### Trigger
```sql
CREATE OR REPLACE FUNCTION public.trigger_normalize_phone()
RETURNS trigger
LANGUAGE plpgsql
AS $$
BEGIN
  NEW.normalized_phone := public.normalize_phone_e164(NEW.phone);
  RETURN NEW;
END;
$$;

DROP TRIGGER IF EXISTS trg_students_normalize_phone ON public.students;
CREATE TRIGGER trg_students_normalize_phone
  BEFORE INSERT OR UPDATE OF phone
  ON public.students
  FOR EACH ROW
  EXECUTE FUNCTION public.trigger_normalize_phone();
-- Repetir pra student_imports + wa_group_members
```

#### VIEW canônica
```sql
CREATE OR REPLACE VIEW public.v_students_unified AS
SELECT
  s.id                                                AS student_id,
  s.normalized_phone,
  s.cohort_id,
  COALESCE(s.name, wgm.wa_name, si.name)              AS canonical_name,
  s.is_mentor,
  s.active,

  -- AC import context
  si.product                                          AS ac_product,
  si.source                                           AS ac_source,
  si.created_at                                       AS ac_imported_at,

  -- WA group context
  wgm.group_id                                        AS wa_group_id,
  wgm.synced_at                                       AS wa_synced_at,

  -- Raw originals (debug/audit)
  s.phone                                             AS student_phone_raw,
  si.phone                                            AS import_phone_raw,
  wgm.phone                                           AS wa_phone_raw
FROM public.students s
LEFT JOIN public.student_imports si
  ON s.normalized_phone = si.normalized_phone
  AND s.cohort_id = si.cohort_id
LEFT JOIN public.wa_group_members wgm
  ON s.normalized_phone = wgm.normalized_phone
  AND s.cohort_id = wgm.cohort_id
WHERE s.normalized_phone IS NOT NULL;

COMMENT ON VIEW public.v_students_unified IS
  'Canonical view de aluno por (normalized_phone, cohort_id). Source-of-truth: students. Mescla audit (student_imports) + mirror WA (wa_group_members). Ref ADR-021.';
```
