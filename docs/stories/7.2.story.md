## Descrição

Como **administrador**, quero poder enviar uma mensagem de teste para um número fixo antes de disparar uma notificação WhatsApp para o grupo — com as variáveis (`{{cohort_name}}`, `{{zoom_link}}`, etc.) interpoladas com dados reais do banco — para garantir que o formato e o conteúdo estão corretos antes de atingir os destinatários reais.

**Contexto técnico:** O preview atual em `admin.html` exibe o template com os placeholders literais (`{{variáveis}}`), não com os valores reais. O botão "Testar" deve chamar a Evolution API com os dados reais do cohort/turma selecionado, mas enviando apenas para o número fixo de teste `554399250490` (Igor - admin), nunca para o grupo ou destinatários reais.

---

## Acceptance Criteria

- [x] **AC1 — Botão "Testar":** o formulário de envio de notificação tem um botão "Testar" separado do botão "Enviar"
- [x] **AC2 — Destinatário fixo:** o test send vai **sempre** para `554399250490` — o número de destino real é ignorado neste fluxo, hardcoded no backend
- [x] **AC3 — Interpolação real:** as variáveis (`{{cohort_name}}`, `{{class_name}}`, `{{zoom_link}}`, `{{mentor_name}}`, `{{class_time_start}}`) são substituídas pelos valores reais do cohort/turma selecionado no formulário
- [x] **AC4 — Feedback visual:** após test send, exibir badge "Teste enviado para +55 43 99250-490" (success) ou mensagem de erro (failure) — sem fechar o formulário
- [x] **AC5 — Sem efeito colateral:** o test send não registra na tabela `notifications` como envio real; pode usar flag `test: true` no payload ou tabela separada
- [x] **AC6 — Número hardcoded no backend:** o número de teste não é configurável via UI — está fixo na edge function ou no servidor para prevenir redirecionamento acidental
