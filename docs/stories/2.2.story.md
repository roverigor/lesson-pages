## Descrição

Como **coordenador da Academia Lendária**, preciso de uma interface no painel admin para criar, visualizar e gerenciar agendamentos de notificações WhatsApp — para que eu possa configurar uma vez e o sistema envie automaticamente toda semana, sem intervenção manual.

**Contexto:** Story 2.1 criou a infraestrutura (tabela + pg_cron). Esta story expõe o gerenciamento via UI na aba "Agendamentos" já existente em `calendario/admin.html` (ou nova aba no view-toggle).

---

## Acceptance Criteria

### AC1 — Nova aba "Agendamentos" no painel
- **Dado** que um admin acessa `calendario/admin.html`
- **Quando** clicar na aba "Agendamentos" no view-toggle
- **Então** deve ver a seção de gerenciamento de agendamentos (visível apenas para admins)

### AC2 — Formulário de criação de agendamento
- **Quando** o admin clicar em "+ Novo Agendamento"
- **Então** deve aparecer um formulário com:
  - Seleção de Classe (dropdown de `classes`)
  - Seleção de Cohort (dropdown de `cohorts`)
  - Tipo de notificação (`class_reminder`, `group_announcement`, `custom`)
  - Tipo de destino (`group`, `individual`, `both`)
  - Antecedência em horas (input numérico, default 2)
  - Template de mensagem (textarea, pré-preenchido conforme tipo)
  - Botões "Salvar" e "Cancelar"

### AC3 — Lista de agendamentos ativos
- **Dado** que existem schedules na tabela
- **Quando** a aba for carregada
- **Então** deve exibir lista com: classe, cohort, tipo, antecedência, próximo disparo (`next_fire_at`), status (ativo/pausado)

### AC4 — Pausar e reativar agendamento
- **Dado** que um schedule existe
- **Quando** o admin clicar em "Pausar" / "Ativar"
- **Então** `active` deve ser atualizado e o botão deve refletir o novo estado

### AC5 — Deletar agendamento
- **Dado** que um schedule existe
- **Quando** o admin clicar em "Deletar" e confirmar
- **Então** o registro deve ser deletado e removido da lista

### AC6 — Próximo disparo visível
- **Dado** que um schedule tem `next_fire_at` calculado
- **Quando** a lista for exibida
- **Então** deve mostrar a data/hora do próximo disparo no fuso America/Sao_Paulo

### AC7 — Zero regressão
- **Quando** a aba de Agendamentos for adicionada
- **Então** Avisos (EPIC-001), Calendário, Cadastro, Turmas e Zoom devem funcionar como antes

## Dev Notes

### Padrão de UI
Seguir exatamente o mesmo padrão de `staff-view` e `notify-view` em `admin.html`:
- Background `#111`, border `#1e1e1e`, border-radius `12px`
- Labels `font-size:11px; color:#555`
- Inputs: classe `login-field` ou `substitute-select`
- Botão principal: classe `btn-save`
- Botão destrutivo: classe `att-btn delete`

### Query principal
```javascript
const { data: schedules } = await sb
  .from('notification_schedules')
  .select('*, classes(name), cohorts(name)')
  .order('created_at', { ascending: false });
```

### Formatar next_fire_at
```javascript
new Date(s.next_fire_at).toLocaleString('pt-BR', {
  timeZone: 'America/Sao_Paulo',
  day: '2-digit', month: '2-digit',
  hour: '2-digit', minute: '2-digit'
})
```
