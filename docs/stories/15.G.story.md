## O que fazer

Implementar trigger automático de versioning: editar `survey_questions` de uma survey que JÁ TEM disparos cria nova `survey_versions` row + atualiza `surveys.current_version_id`. UI mostra badge "X disparos usaram versão atual" na lista de forms.

## Business Value

- **FR-13 versionamento:** edita form sem invalidar links já enviados
- **NFR-15 reliability:** test E2E garante alunos antigos respondem v1, novos respondem v2
- **AC-10 spec passa:** versão isolada preserva integridade respostas
- **CS confiança:** edita form sem medo de quebrar disparos ativos

## Dev Notes

### Trigger SQL Skeleton

```sql
CREATE OR REPLACE FUNCTION auto_version_survey()
RETURNS TRIGGER AS $$
DECLARE
  v_survey_id UUID;
  v_dispatched_count INT;
  v_new_version_id UUID;
  v_new_version_num INT;
BEGIN
  v_survey_id := COALESCE(NEW.survey_id, OLD.survey_id);

  SELECT COUNT(*) INTO v_dispatched_count
  FROM survey_links
  WHERE survey_id = v_survey_id AND send_status = 'sent';

  IF v_dispatched_count > 0 THEN
    -- Create new version
    SELECT COALESCE(MAX(version_number), 0) + 1 INTO v_new_version_num
    FROM survey_versions WHERE survey_id = v_survey_id;

    INSERT INTO survey_versions (survey_id, version_number) VALUES (v_survey_id, v_new_version_num)
    RETURNING id INTO v_new_version_id;

    UPDATE surveys SET current_version_id = v_new_version_id WHERE id = v_survey_id;

    -- Notify (audit)
    RAISE NOTICE 'Survey % auto-versioned to v%', v_survey_id, v_new_version_num;
  END IF;

  RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_auto_version_survey
  BEFORE INSERT OR UPDATE OR DELETE ON survey_questions
  FOR EACH ROW EXECUTE FUNCTION auto_version_survey();
```
