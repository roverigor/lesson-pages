## Testing Strategy

### Validation Gates

| Gate | Como Validar |
|---|---|
| OQ-1..6, 8, 11 (BLOCKING) | Cada um deve ter resposta concreta + evidência (URL, screenshot, payload) em research.json |
| Schema research.json | Validador JSON contra schema inline (T6.1) |
| AC webhook real | Disparar 1 evento sandbox AC → confirmar payload chega + estrutura conforme oq2 |
| AC custom_field write | curl com token real → confirmar HTTP 200 + custom field aparece no contact |
| Meta templates | Listar via Meta Graph API: `GET /{WHATSAPP_BUSINESS_ACCOUNT_ID}/message_templates` |
| Repo cs-portal | `gh repo view roverigor/cs-portal` retorna 200 + lista colaboradores match CS team |
| CODEOWNERS | PR teste em supabase/migrations/* dispara auto-assign |
| Meta template approval | Confirmação Meta no email/dashboard (pode demorar 24-48h) |

### Edge Cases para Validar

- AC pode reenviar mesmo evento? Confirmar comportamento real (afeta ASM-5/EC-1)
- AC envia múltiplos products na mesma compra? (afeta EC-24)
- Meta template aprovação leva quanto tempo na prática? (afeta CRIT-6 prazo go-live)

### Não-Testáveis Nesta Story

- Idempotência sob carga real (Story 15.B)
- Cold start mitigation funcionando em prod (Story 15.E observabilidade)
- Form builder UX (Story 15.6)
