## Descrição

Como **coordenador**, quero que a presença dos professores, hosts e mentores seja automaticamente registrada no relatório do sistema a partir dos dados do Zoom — para que eu não precise entrar no admin e aplicar manualmente a cada aula.

### Contexto Técnico

- `zoom_participants` já tem fuzzy match com mentores via `mark_mentor_participants()`
- `student_attendance` recebe presença de alunos via `transfer_to_attendance`
- A tabela `attendance` (calendário) aceita registros de staff
- O frontend tem `applyStaffAttendanceFromZoom()` manual — precisa virar automático
- O pipeline diário (`daily_pipeline`) roda às 03:00 BRT e já faz import + match + transfer para alunos

---

## Acceptance Criteria

- [ ] **AC1 — Step automático no pipeline:** Após `transfer_to_attendance` (alunos), o `daily_pipeline` executa um novo step `sync_staff_attendance` que marca presença de mentores
- [ ] **AC2 — Match mentor → Zoom:** Busca em `zoom_participants` registros onde `participant_name` faz match com nomes/aliases de `mentors` ativos
- [ ] **AC3 — Registro em attendance:** Para cada mentor matched, insere registro na tabela `attendance` com `source = 'zoom'`, `class_date`, `mentor_id`, `cohort_id`, `zoom_meeting_id`
- [ ] **AC4 — Anti-duplicata:** UPSERT por `(mentor_id, class_date, zoom_meeting_id)` — não duplica se rodar novamente
- [ ] **AC5 — Log no automation_runs:** Step logado com `p_step_name = 'sync_staff_attendance'`, contagem de mentores processados/matched
- [ ] **AC6 — Sem impacto em fluxo manual:** O botão "Aplicar Presença Staff" no admin continua funcionando independentemente

---

## Scope

### IN
- Nova action `sync_staff_attendance` na edge function `zoom-attendance`
- Integrar como step final do `daily_pipeline`
- RPC ou query direta para inserir em `attendance`

### OUT
- Alterações na UI do admin (mantém botão manual como está)
- Alterações no schema de `attendance` (usa estrutura existente)
- Match por email (fase futura — hoje é por nome/alias)
