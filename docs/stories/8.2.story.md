## Descrição

Como **coordenador acessando pelo celular**, quero que o painel de presença (`/presenca`) exiba participantes como cards empilhados — com o botão "Vincular" sempre visível — para poder fazer vinculações sem scroll horizontal.

**Contexto técnico:** O painel `/presenca/index.html` usa tabelas HTML (`<table>`) para listar participantes de reuniões Zoom. Em mobile, essas tabelas causam scroll horizontal, tornando impossível completar a ação de vincular. A solução é converter as tabelas em listas de cards via CSS responsivo.

---

## Acceptance Criteria

- [ ] **AC1 — Cards em mobile:** em telas ≤768px, cada linha de participante vira um card empilhado com: nome, duração, status de vínculo, botão de ação
- [ ] **AC2 — Botão sempre visível:** o botão "Vincular" ou "Vinculado" fica visível sem scroll horizontal em qualquer tela ≥320px
- [ ] **AC3 — Pesquisa funciona:** o campo de busca de participantes funciona normalmente em mobile (sem teclado sobrepondo o campo)
- [ ] **AC4 — Modal de busca de aluno:** o modal de busca para vincular é usável em mobile (ocupa ≥90% da largura, não transborda)
- [ ] **AC5 — Sem regressão desktop:** em telas >768px, comportamento atual preservado

---

## Tasks

- [ ] Identificar todas as tabelas em `/presenca/index.html` que listam participantes
- [ ] Adicionar CSS `@media (max-width: 768px)` — `table, thead, tbody, tr, th, td { display: block; }` com layout de card
- [ ] Garantir que botão de ação (Vincular/Vinculado) é o último elemento de cada card e tem largura 100% em mobile
- [ ] Ajustar modal de busca de aluno: `width: min(90vw, 500px)`, `max-height: 80vh`, scroll interno
- [ ] Testar em 375px — vincular um participante do início ao fim sem scroll horizontal
