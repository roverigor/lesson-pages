## Riscos não-cobertos

Production migration tem 8 listados — sólido baseline. **Faltam:**

1. **R9 — `is_dashboard_admin()` retorna NULL pra session sem JWT claim de admin** — diferente de "false negative pra admin legítimo" (já em R6). Se função retornar NULL, comparação `OR public.is_dashboard_admin()` resulta NULL em USING → linha não-visível (semântica Postgres). Edge case raro mas real.

2. **R10 — Locks de schema durante ALTER TABLE bloqueiam writes simultâneos em prod** — `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` adquire `ACCESS EXCLUSIVE` lock. 28 tabelas × write traffic NPS/lembretes pode causar lock storm em janela fora de madrugada. Mitigação implícita em Janela de Manutenção mas não enumerada como risco.

3. **R11 — Audit trail circular** — migration que protege `audit_log` precisa escrever em `audit_log` (per AC8). Se policy é aplicada antes do insert log, pode bloquear self-insert. Ordering crítico: `ALTER ENABLE RLS audit_log` por último OU insert SEM RLS via service_role.

4. **R12 — Edge functions com SUPABASE_ANON_KEY descoberta em produção (não staging)** — Dev Notes §3 + Risco #8 mencionam grep, mas se edge function usar anon key intermitentemente (ex: `if env.A else B` fallback), staging pode passar e prod quebrar. Não há AC verificando isso em smoke prod.
