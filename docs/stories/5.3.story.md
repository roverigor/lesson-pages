## Descrição

Como **admin da Academia Lendária**, quero ver uma lista de nomes do Zoom que não foram reconhecidos automaticamente e provavelmente são mentores, para que eu possa vinculá-los ao perfil correto com um clique — e essa vinculação seja aplicada retroativamente a todos os meetings anteriores.

**Contexto técnico:** Depende da Story 5.1 (action `zoom_mentor_candidates`). A vinculação salva o nome no `aliases[]` do mentor (via Story 5.2's addAlias) e chama `mark_mentor_participants()` via RPC para propagar retroativamente.

## Technical Notes

```javascript
// Chamar action zoom_mentor_candidates
async function loadZoomCandidates() {
  const res = await fetch(ZOOM_ATTENDANCE_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${SUPABASE_ANON_KEY}` },
    body: JSON.stringify({ action: 'zoom_mentor_candidates' })
  });
  const { candidates } = await res.json();
  return candidates; // [{ name, meeting_count }]
}

// Vincular e propagar
async function linkCandidateToMentor(candidateName, mentorId, currentAliases) {
  await addAlias(mentorId, currentAliases, candidateName);
  const { data } = await sb.rpc('mark_mentor_participants');
  return data?.[0]?.updated_count ?? 0;
}
```

## Risk Notes

- **Risk:** `mark_mentor_participants()` retroativo pode fazer matching incorreto se alias for ambíguo
- **Mitigação:** O admin confirma explicitamente antes de vincular; o alias é visível e removível via Story 5.2
