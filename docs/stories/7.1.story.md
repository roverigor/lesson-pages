## Descrição

Como **administrador da Academia Lendária**, quero que todas as ações destrutivas irreversíveis no painel admin (delete de turma, mentor, cohort, agendamento) exijam uma confirmação explícita antes de executar — para evitar perda acidental de dados em produção.

**Contexto técnico:** Hoje o painel `/calendario/admin.html` executa deletes diretamente no `onclick` de botões de lixeira sem nenhum modal intermediário. Uma confirmação baseada em `confirm()` nativo do browser ou um modal customizado resolve o problema com mínimo de código.

---

## Acceptance Criteria

- [ ] **AC1 — Confirmação universal:** qualquer ação de delete no admin (turma, mentor, agendamento, notificação agendada) exige confirmação antes de prosseguir
- [ ] **AC2 — Modal customizado:** a confirmação usa um modal HTML (não `window.confirm()`) com texto descritivo: "Deletar **{nome}**? Esta ação não pode ser desfeita."
- [ ] **AC3 — Botões claros:** modal tem botão "Cancelar" (fecha sem ação) e "Deletar" (cor vermelha, executa)
- [ ] **AC4 — Cancelar não apaga:** clicar em "Cancelar" ou fora do modal não executa nenhuma operação de delete
- [ ] **AC5 — Sem regressão:** todas as ações não-destrutivas (editar, salvar, fechar) continuam funcionando sem modal extra

---

## Tasks

- [ ] Criar componente `showDeleteConfirm(name, onConfirm)` — função JS reutilizável que exibe modal de confirmação e chama `onConfirm()` apenas se usuário confirmar
- [ ] Substituir todos os `onclick` de delete direto em `admin.html` para chamar `showDeleteConfirm(...)` antes da operação
- [ ] Estilizar modal com design system existente (dark, borda vermelha no botão de confirmação)
- [ ] Testar: delete de turma, mentor, agendamento — confirmar que apenas após "Deletar" a ação ocorre
