## Story
Como **administrador da Academia Lendária**,
Eu preciso **investigar TODOS os crons NPS atuais (ativos + dormentes + comentados), classificar cada um como Intencional ou Esquecido, e estabelecer UI button gate humano pra qualquer reativação (NUNCA cron silencioso)**,
Para que **nenhum cron NPS dispare envios em prod sem autorização literal user explícita (per CLAUDE.md "Comunicação Externa NON-NEGOTIABLE")**.

## Contexto

Migrations contêm `-- SELECT cron.schedule(...)` comentados em jobs NPS (exemplo: `20260517010400_dispatch_class_nps_cron.sql`). Estado atual ambíguo:

- **Cenário A — Dormante intencional:** comentário foi decisão consciente (gate humano + risco prod)
- **Cenário B — Esquecido:** comentário deixado durante refactor, sem decisão arquitetural

**Risco crítico:** Se for B e alguém descomentar sem revisar → disparo massivo pra alunos prod sem aprovação. Violação CLAUDE.md "Comunicação Externa NON-NEGOTIABLE".

**Sistema EM PRODUÇÃO** com 1k+ alunos ativos. Aluno recebe Meta DM cada envio NPS. Reativação acidental = problema operacional + reputational + custo template Meta.

**Memory cross-ref:** `slack-always-on-dispatchers` — toda dispatcher precisa Slack startup + final notify.

## Acceptance Criteria (TODOS NON-NEGOTIABLE per CLAUDE.md)

### AC1 — ADR lista TODOS crons NPS
**Given** investigação completa
**When** ADR-025 criado
**Then** contém:
- TODOS pg_cron jobs com `command ILIKE '%nps%' OR jobname ILIKE '%nps%'` (ativos)
- TODOS migrations com `-- SELECT cron.schedule` comentado relacionado a NPS
- TODOS jobs `active=false` em cron.job
- TODOS `cron.unschedule(...)` chamados em migrations
- Classificação **por cron**: Intencional | Esquecido | A decidir

### AC2 — Relatório dispatches NPS últimos 30d
**Given** logs prod
**When** report gerado
**Then** anexado ao ADR-025:
- Dispatches NPS esperados (baseado em crons ativos)
- Dispatches NPS efetivos (audit_log + dispatch_history)
- Diff esperado vs efetivo (anomalias)

### AC3 — UI button gate pra reativação
**Given** decisão "reativar cron" pra qualquer cron
**When** reativação ocorre
**Then** EXISTE UI button explícito em `/admin/dispatch` (não cron silencioso)
**E** botão exige clique humano + preview destinatários + content template
**E** ZERO reativação via cron schedule sem UI

### AC4 — Preview obrigatório (lista + conteúdo)
**Given** clique no UI button (AC3)
**When** request inicia
**Then** UI mostra:
- Lista completa de destinatários (count + amostra ~10 rows)
- Conteúdo do template (Meta DM body)
- Botões "Confirmar" + "Cancelar"

### AC5 — Confirmação modal com count + sample
**Given** preview render (AC4)
**When** user clica "Confirmar"
**Then** modal aparece com:
- Count total destinatários
- Sample 5 destinatários por nome
- Confirmação dupla "Sim, enviar pra <N> alunos"
- Botão "Cancelar" sempre disponível

### AC6 — Audit log dispatch manual
**Given** dispatch executa (pós-confirmação)
**When** envio inicia
**Then** INSERT em `audit_log` event_type='nps_manual_dispatch' com payload:
- operator (admin email)
- timestamp
- recipients_count
- template_name + template_version
- dry_run flag (boolean)

### AC7 — Modo dry-run disponível
**Given** preview render (AC4)
**When** user marca checkbox "Dry-run"
**Then** UI envia 0 mensagens reais
**E** Audit log entry registrada normalmente com `dry_run=true`
**E** Logs completos do que seria enviado pra Slack #devops

### AC8 — Slack alert startup + final
**Given** dispatch real executa (não dry-run)
**When** envia primeira message AND última message
**Then** Slack #devops recebe:
- Startup: "NPS manual dispatch START — operator=<>, recipients=<N>"
- Final: "NPS manual dispatch COMPLETE — sent=<N>, failed=<M>, duration=<s>"

### AC9 — PROIBIDO re-ativar cron silencioso
**Given** investigação conclui qualquer cron NPS = "reativar"
**When** PR mergeado
**Then** PR NÃO contém `cron.schedule` ativo (não-comentado) em migrations NPS
**E** ADR-025 contém citação textual de mensagem user autorizando reativação
**E** Mecanismo de reativação = UI button (AC3), não cron

### AC10 — Comment SQL explícito (dormante intencional)
**Given** cron classificado "Dormante intencional"
**When** migration original referenciada
**Then** comment SQL adicionado: `-- DORMANT BY DESIGN ref ADR-025. Do NOT uncomment without re-running gate review per CLAUDE.md "Comunicação Externa"`
