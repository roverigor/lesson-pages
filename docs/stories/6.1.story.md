## Descrição

Como **admin da Academia Lendária**, quero que ao terminar uma reunião no Zoom os participantes sejam importados, matcheados e registrados em `student_attendance` automaticamente — sem nenhuma ação manual — para que a presença esteja disponível em minutos após o fim de cada aula.

**Contexto técnico:** O webhook `meeting.ended` já é recebido pela edge function `zoom-webhook` e atualiza `zoom_host_sessions`. O que falta é, após esse evento, disparar automaticamente a importação de participantes (equivalente ao fluxo manual em `/calendario/admin`). A estratégia é: `zoom-webhook` insere a reunião numa fila de processamento (tabela `zoom_import_queue`), e um pg_cron processa essa fila chamando a edge function `zoom-attendance` a cada 2 minutos.

---

## Acceptance Criteria

- [ ] **AC1 — Trigger automático:** quando `meeting.ended` chega, a reunião é inserida em `zoom_import_queue` com status `pending`
- [ ] **AC2 — Worker pg_cron:** job `zoom-auto-import` roda a cada 2 minutos, pega reuniões `pending` da fila, chama `zoom-attendance` com `meeting_id`, marca como `done` ou `error`
- [ ] **AC3 — Idempotência:** se a reunião já foi importada antes (zoom_meetings.processed = true), o worker ignora e marca como `skipped`
- [ ] **AC4 — Fallback manual intacto:** o fluxo manual em `/calendario/admin` continua funcionando sem alterações
- [ ] **AC5 — Log de erros:** se a importação falhar, registrar o erro em `zoom_import_queue.error_message` e não bloquear o processamento das demais reuniões
- [ ] **AC6 — Retry automático:** reuniões com status `error` são retentadas até 3 vezes (campo `attempt_count`); após 3 falhas, ficam em `failed` para revisão manual
- [ ] **AC7 — Visibilidade no admin:** `/calendario/admin` exibe o status da fila (quantas pendentes, última importação automática, eventuais erros)

## Technical Notes

```sql
-- Tabela de fila
CREATE TABLE zoom_import_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  meeting_id TEXT NOT NULL,
  zoom_uuid TEXT,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'processing', 'done', 'skipped', 'failed', 'error')),
  attempt_count INT DEFAULT 0,
  error_message TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  processed_at TIMESTAMPTZ
);

-- pg_cron (a cada 2 minutos)
SELECT cron.schedule(
  'zoom-auto-import',
  '*/2 * * * *',
  $$
    SELECT net.http_post(
      url := current_setting('app.zoom_attendance_url'),
      body := json_build_object('meeting_id', meeting_id)::jsonb,
      headers := json_build_object(
        'Authorization', 'Bearer ' || current_setting('app.supabase_service_key'),
        'Content-Type', 'application/json'
      )::jsonb
    )
    FROM zoom_import_queue
    WHERE status = 'pending' AND attempt_count < 3
    LIMIT 5;
  $$
);
```

## Risk Notes

- **Risk:** pg_net pode não ter acesso à edge function se URL não estiver configurada como setting
- **Mitigação:** Configurar `app.zoom_attendance_url` via `ALTER DATABASE ... SET`; testar antes do deploy

- **Risk:** Reunião processada antes de todos os participantes saírem (Zoom demora alguns minutos para gerar o relatório completo)
- **Mitigação:** Aguardar 5 minutos após `meeting.ended` antes de enfileirar (campo `process_after TIMESTAMPTZ = now() + interval '5 min'`)
