# Story 4.5 — Dashboard /avaliacao/index.html conectado ao DB

**Epic:** EPIC-004  
**Status:** Ready  
**Type:** Frontend  
**Depende de:** Story 4.1, 4.3

## O que fazer

Migrar `/avaliacao/index.html` de dados hardcoded para leitura dinâmica do banco, mantendo layout e UX existentes. O dashboard exige admin auth (mesma sessão do painel admin).

## Acceptance Criteria

- [ ] Dados carregados do Supabase ao invés do objeto `DATA` hardcoded
- [ ] Para cada survey `status='active'` ou `status='closed'`: exibe card na overview grid
- [ ] NPS score calculado dinamicamente: ((promotores - detratores) / total) * 100
- [ ] CSAT score: média das notas
- [ ] "N respondentes" calculado do banco
- [ ] Comentários listados do banco (campo `feedback` de `student_nps`)
- [ ] Se não há surveys no banco → mensagem "Nenhuma avaliação realizada ainda"
- [ ] Mantém todos os estilos/cards/animações existentes
- [ ] Requer admin auth: usa mesma chave anon + verifica sessão ativa (ou apenas admin key)
- [ ] Loading state enquanto busca dados

## Nota de migração

O `DATA` hardcoded pode ser removido após a story ou mantido como fallback para histórico pré-sistema. Decisão: **remover** — o histórico do Tally pode ser importado via migration seed se necessário.
