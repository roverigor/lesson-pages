## Dev Notes

### Nginx Config Skeleton

```nginx
# Adicionar dentro do server block existente em
# /etc/nginx/sites-available/painel-lesson-pages

server {
  listen 443 ssl http2;
  server_name painel.igorrover.com.br;

  # SSL existente - não tocar
  ssl_certificate /etc/letsencrypt/live/painel.igorrover.com.br/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/painel.igorrover.com.br/privkey.pem;

  # Painel admin existente
  location / {
    proxy_pass http://localhost:3080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  # ── EPIC-015 Story 15.K — CS Portal ──
  location /cs/ {
    proxy_pass http://localhost:3081/;  # trailing slash strips /cs/ from path
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    access_log /var/log/nginx/cs-portal-access.log;
    error_log /var/log/nginx/cs-portal-error.log;
  }

  # Redirect /cs (sem trailing) → /cs/
  location = /cs {
    return 301 /cs/;
  }
}
```

### Dockerfile cs-portal nginx Internal Routing

Container já configurado para servir `public/dist/` + fallback `index.html` (Story 15.J). Path mapping no nginx VPS:
- Request: `https://painel.igorrover.com.br/cs/dashboard`
- Nginx VPS strip `/cs/` → request interno: `http://localhost:3081/dashboard`
- Container nginx: try_files → fallback `index.html` (SPA hash routing client-side)
