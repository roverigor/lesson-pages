## O que fazer

Refactor de `dispatch-survey` (existente) para aceitar lista explícita de `student_ids` (não apenas filtrar por cohort_id). Implementar UI no cs-portal `/cs/dispatch` com 3 modos: cohort inteira, alunos avulsos (multi-select search), misto.

Inclui captura de `meta_message_id` na response Meta + chamada `ac-report-dispatch` callback async.

- **Backward compat EPIC-004 preservada:** feature flag `EPIC015_NEW_DISPATCH` permite rollback

## Dev Notes

### dispatch-survey Refactor

```typescript
// Antes
if (survey.cohort_id) { /* fetch students by cohort */ }

// Depois
let students: Student[] = [];
if (body.student_ids?.length) {
  // Modo explícito (NEW)
  const { data } = await client.from("students")
    .select("id, name, phone, ac_contact_id")
    .in("id", body.student_ids)
    .eq("active", true);
  students = data ?? [];
} else if (body.cohort_id || survey.cohort_id) {
  // Modo legacy (compatible)
  const cohortId = body.cohort_id || survey.cohort_id;
  const { data } = await client.from("students").select(...).eq("cohort_id", cohortId);
  students = data ?? [];
}
```

### Captura meta_message_id

```typescript
const result = await sendWhatsAppTemplate(...);
if (result.success && result.messageId) {
  await client.from("survey_links")
    .update({ send_status: "sent", sent_at: new Date().toISOString(), meta_message_id: result.messageId })
    .eq("id", pl.id);

  // Fire-and-forget callback AC
  if (student.ac_contact_id) {
    fetch(`${SUPABASE_URL}/functions/v1/ac-report-dispatch`, {
      method: "POST",
      headers: { Authorization: `Bearer ${SUPABASE_SERVICE_ROLE_KEY}` },
      body: JSON.stringify({ link_id: pl.id, ac_contact_id: student.ac_contact_id, custom_field_id: AC_CUSTOM_FIELD_ID, value: pl.token })
    }).catch(err => console.error("AC callback fire-and-forget failed:", err));
  }
}
```
