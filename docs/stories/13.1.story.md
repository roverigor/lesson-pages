## Descrição

Como **coordenador**, quero que ao editar a equipe de uma turma, apenas os membros alterados sejam afetados — para que trocar um mentor não feche/reabra todos os registros e não haja risco de perder dados.

---

## Acceptance Criteria

- [x] **AC1 — Diff por membro:** `saveClassV2()` compara a equipe atual no DB (registros com `valid_until IS NULL`) com a equipe no formulário. Identifica: membros mantidos, membros removidos, membros adicionados
- [x] **AC2 — Membro mantido:** Se um membro (mesmo `mentor_id` + `role` + `weekday`) existe no DB e no formulário, o registro NÃO é alterado
- [x] **AC3 — Membro removido:** Se um membro existe no DB mas não no formulário, recebe `valid_until = hoje` (preserva histórico)
- [x] **AC4 — Membro adicionado:** Se um membro existe no formulário mas não no DB, é inserido com `valid_from = hoje, valid_until = NULL`
- [x] **AC5 — Sem auto-close de ciclo:** Remover toda a lógica de auto-close (`latestClosedUntil`, `hasActiveBeforeToday`, auto-fecha ciclo com `yesterday`)
- [x] **AC6 — Edição idempotente:** Salvar a turma sem alterar a equipe não cria novos registros nem modifica os existentes
- [x] **AC7 — Calendário preservado:** `buildEventsFromDB()` continua funcionando sem alterações — o calendário mostra mentores corretos por data

## Technical Notes

### Lógica do diff

```javascript
// Chave de comparação: mentor_id + role + weekday
function mentorKey(cm) {
  return `${cm.mentor_id}|${cm.role}|${cm.weekday}`;
}

// dbActive = registros atuais com valid_until IS NULL
// formMembers = registros montados a partir do formulário

const dbKeys = new Set(dbActive.map(mentorKey));
const formKeys = new Set(formMembers.map(mentorKey));

const kept = dbActive.filter(cm => formKeys.has(mentorKey(cm)));     // nada a fazer
const removed = dbActive.filter(cm => !formKeys.has(mentorKey(cm))); // valid_until = hoje
const added = formMembers.filter(cm => !dbKeys.has(mentorKey(cm)));  // insert com valid_from = hoje
```
