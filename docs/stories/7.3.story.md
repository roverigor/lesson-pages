## Descrição

Como **mentor da Academia Lendária**, quero que minha sessão no portal `/equipe` expire automaticamente após 8 horas de inatividade — e que o logout limpe minha autenticação de forma segura — para que minha conta não fique acessível em dispositivos compartilhados ou esquecida aberta.

**Contexto técnico:** Hoje a autenticação em `/equipe/index.html` usa `localStorage` com `mentor_id`, `mentor_name`, `mentor_role`, `mentor_phone` salvos sem expiração. O login é feito via RPC `authenticate_mentor(p_phone, p_password)` que retorna dados do mentor. A melhoria é adicionar um timestamp de expiração no `localStorage` e verificar a validade a cada ação relevante — **sem** migrar para Supabase Auth completo (que quebraria o fluxo de RPC customizado). A solução mantém o RPC mas adiciona lógica de TTL client-side + renovação automática.

**Nota de escopo:** Migração completa para Supabase Auth JWT está fora do escopo desta story por risco de regressão. A solução de TTL client-side é incremental e segura.

---

## Acceptance Criteria

- [x] **AC1 — Expiração de 8h:** ao fazer login, salvar timestamp `mentor_session_expires` em `localStorage` com valor `now() + 8h`
- [x] **AC2 — Verificação automática:** ao carregar `/equipe` e antes de qualquer operação de dados, verificar se `mentor_session_expires > now()`; se expirado, redirecionar para login com mensagem "Sua sessão expirou. Faça login novamente."
- [x] **AC3 — Renovação por atividade:** qualquer ação do mentor (check-in, salvar alias, clicar tab) renova o timestamp por mais 8h a partir da última ação
- [x] **AC4 — Logout seguro:** ao clicar "Sair", limpar todos os campos do mentor em `localStorage` (`mentor_id`, `mentor_name`, `mentor_role`, `mentor_phone`, `mentor_session_expires`) e redirecionar para o formulário de login
- [x] **AC5 — Sem regressão:** o fluxo de login via RPC `authenticate_mentor` continua funcionando sem alterações
- [x] **AC6 — Mensagem clara na expiração:** quando redireciona para login por expiração, exibir banner informativo (não só campo vazio)

## Tasks

- [x] Criar função `setMentorSession(mentorData)` — salva dados + `mentor_session_expires = Date.now() + 8*60*60*1000`
- [x] Criar função `getMentorSession()` — retorna dados se válido, `null` se expirado (e limpa localStorage)
- [x] Criar função `renewMentorSession()` — atualiza `mentor_session_expires` para `now + 8h`
- [x] Substituir todos os `localStorage.setItem` e `localStorage.getItem` de sessão em `equipe/index.html` pelas novas funções
- [x] Adicionar verificação de expiração no início do `DOMContentLoaded` em `equipe/index.html`
- [x] Adicionar `renewMentorSession()` nos event listeners de ações principais (check-in, salvar alias)
- [x] Exibir banner "Sessão expirada" na tela de login quando redirecionado por expiração (via query param `?expired=1`)
