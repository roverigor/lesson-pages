## Descrição

Como **mentor da Academia Lendária**, quero que o sistema envie automaticamente uma mensagem de WhatsApp ao aluno quando ele faltar 2 aulas consecutivas — sem precisar monitorar manualmente a lista de presença — para que a intervenção aconteça de forma proativa e no momento certo.

**Contexto técnico:** A Evolution API já está integrada no sistema (EPIC-001). Os dados de presença estão em `student_attendance`. Esta story cria um pg_cron job que detecta ausências consecutivas e dispara mensagens via Evolution API, com controle para não spam (máximo 1 mensagem por ciclo de ausência por aluno).

## Acceptance Criteria

- [ ] **AC1 — Detecção de faltas consecutivas:** pg_cron job `zoom-absence-alert` detecta alunos com 2+ faltas consecutivas nas últimas aulas da turma
- [ ] **AC2 — Mensagem personalizada:** mensagem enviada inclui nome do aluno, nome da turma e quantas aulas foram perdidas
- [ ] **AC3 — Anti-spam:** cada ciclo de ausência gera no máximo 1 mensagem. Somente nova sequência de faltas (após o aluno comparecer) dispara novo alerta
- [ ] **AC4 — Número de destino correto:** mensagem enviada para o `phone` do aluno cadastrado, no formato Evolution API (554399XXXXXXX)
- [ ] **AC5 — Registro de envio:** cada mensagem enviada é registrada em `zoom_absence_alerts` (student_id, cohort_id, sent_at, message_text, whatsapp_status)
- [ ] **AC6 — Respeita opt-out:** se `students.whatsapp_alerts_enabled = false`, não envia (campo a ser adicionado)
- [ ] **AC7 — Falha silenciosa:** se Evolution API falhar, registrar o erro mas não bloquear o cron job nem alertar o aluno duas vezes

## Technical Notes

```sql
-- Função de detecção de ausências consecutivas sem alerta enviado
CREATE OR REPLACE FUNCTION get_consecutive_absences_needing_alert()
RETURNS TABLE (student_id UUID, student_name TEXT, phone TEXT, cohort_id UUID, cohort_name TEXT, consecutive_count INT)
AS $$
  WITH last_classes AS (
    -- Pegar últimas N datas de aula por cohort
    SELECT DISTINCT class_date, cohort_id FROM student_attendance ORDER BY class_date DESC
  ),
  student_presence AS (
    -- Para cada aluno+turma, marcar presença/ausência por data
    ...
  ),
  consecutive AS (
    -- Contar sequência de ausências mais recente
    ...
  )
  SELECT s.id, s.name, s.phone, c.id, c.name, consecutive.count
  FROM consecutive c
  JOIN students s ON s.id = c.student_id
  WHERE c.count >= 2
    AND s.whatsapp_alerts_enabled = true
    AND NOT EXISTS (
      SELECT 1 FROM zoom_absence_alerts a
      WHERE a.student_id = s.id
        AND a.cohort_id = c.cohort_id
        AND a.consecutive_count = c.count  -- mesmo ciclo
        AND a.sent_at > now() - interval '7 days'
    );
$$ LANGUAGE sql;
```

## Risk Notes

- **Risk:** Aluno com telefone inválido ou não cadastrado — mensagem não pode ser enviada
- **Mitigação:** Validar `phone` não nulo antes de tentar enviar; registrar `error` na tabela

- **Risk:** Aluno considera mensagem invasiva (spam)
- **Mitigação:** Tom cuidadoso no template; campo opt-out `whatsapp_alerts_enabled`; máximo 1 mensagem por ciclo

- **CRÍTICO — Testes:** NUNCA enviar para telefones reais durante desenvolvimento. Usar SEMPRE `554399250490` (número de teste do Igor) para qualquer teste ou debug
