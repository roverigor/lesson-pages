## Descrição

Como **coordenador**, quero que os dados de equipe no banco estejam consistentes — sem registros duplicados, gaps ou órfãos — e que todas as 4 turmas ativas (Cohort Advanced T2, Cohort Fundamentals T4, PS Advanced, PS Fundamentals) mostrem equipe correta no calendário e no card.

---

## Acceptance Criteria

- [x] **AC1 — Limpar duplicatas:** Migration identifica e remove registros duplicados em `class_mentors` (mesmo class_id + mentor_id + role + weekday + valid_from)
- [x] **AC2 — Validar turmas ativas:** Query de validação confirma que todas as 4 turmas ativas têm pelo menos 1 membro com `valid_until IS NULL` em cada weekday configurado
- [x] **AC3 — Calendário correto:** Para cada turma ativa, verificar que `buildEventsFromDB` retorna mentores para datas passadas (histórico) e futuras (equipe atual)

- [x] **AC5 — Migration idempotente:** A migration pode ser executada múltiplas vezes sem efeito colateral

## Validation Query

```sql
-- Verifica que toda turma ativa tem equipe ativa
SELECT c.name, cm.weekday, COUNT(*) as active_members
FROM classes c
JOIN class_mentors cm ON cm.class_id = c.id
WHERE c.active = true AND cm.valid_until IS NULL
GROUP BY c.name, cm.weekday
ORDER BY c.name, cm.weekday;
```
