## Descrição

Como **coordenador**, quero que o lembrete diário de staff considere overrides de agenda (adições e remoções pontuais) — para que se eu adicionar um mentor substituto ou remover alguém de um dia específico, o sistema envie o lembrete corretamente.

### Contexto Técnico

- O frontend `buildEventsFromDB()` aplica `schedule_overrides` após filtrar `class_mentors`
- A function `process_daily_staff_reminders()` (migration 20260416110000) **não consulta** `schedule_overrides`
- Tabela `schedule_overrides` tem campos: `class_id`, `override_date`, `action` ('add'|'remove'), `mentor_id`, `role`, `weekday`
- Discrepância: se coordenador adiciona substituto via override, ele não recebe lembrete; se remove alguém, a pessoa removida ainda recebe

---

## Acceptance Criteria

- [ ] **AC1 — Remoção por override:** Se `schedule_overrides` tem `action = 'remove'` para um `mentor_id` + `class_id` na data de hoje, esse mentor NÃO recebe lembrete
- [ ] **AC2 — Adição por override:** Se `schedule_overrides` tem `action = 'add'` para um `mentor_id` + `class_id` na data de hoje, esse mentor RECEBE lembrete com o role definido no override
- [ ] **AC3 — Companion info atualizada:** A info de companion (Professor vê Host, Host vê Professor) também considera overrides
- [ ] **AC4 — Prioridade override:** Override prevalece sobre `class_mentors` — se mentor está no ciclo ativo MAS tem override de remoção, não recebe
- [ ] **AC5 — Sem impacto em dias normais:** Dias sem override na tabela continuam funcionando exatamente como antes
