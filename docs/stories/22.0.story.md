## Contexto

Hoje "Aula NN" mostrada em `nps_results_by_survey` (e em outros lugares: relatórios MD, dispatchers, dashboards) é **inferida em runtime** contando entries em `zoom_meetings` antes de `session_date`. Lógica frágil — band-aid atual:

```sql
SELECT COUNT(DISTINCT (zm.start_time AT TIME ZONE 'America/Sao_Paulo')::date) + 1
FROM zoom_meetings zm
WHERE zm.class_id = a.class_id AND zm.cohort_id = a.cohort_id
  AND COALESCE(zm.participants_count, 0) >= 10
  AND COALESCE(zm.duration_minutes, 0) >= 60
  AND (zm.start_time AT TIME ZONE 'America/Sao_Paulo')::date < a.session_date
```

**Problemas estruturais identificados (revisão @data-engineer):**

1. **`zoom_meetings` é fonte errada** — tabela registra EXECUÇÃO (webhook Zoom), não PLANEJAMENTO. Acoplar ordem das aulas a webhook é frágil.
2. **Heurísticas quebram em edge cases:**
   - `participants≥10` falha em turma pequena/private cohort legítimo
   - `duration≥60` falha em workshop curto legítimo (45min)
   - Mentor session com 12 alunos passa filtro mas não é aula
3. **Sem snapshot histórico** — cohort remarcar (cancelar Aula 3, virar Aula 4) altera labels retroativamente → quebra audit trail / relatórios históricos
4. **Inconsistência multi-dashboard** — cada lugar reimplementa lógica (nps-results vs envios vs daily MD report)

**Bug recente (2026-05-21):** Advanced T2 mostrava "Aula 09" quando era "Aula 07". Hotfix tático aplicado (`DISTINCT date + filtros heurísticos`) mas é solução temporária.

## Decisão arquitetural (revisada)

**Tabela dedicada `cohort_sessions` (NÃO coluna em `class_cohorts`)**

### Razão de não usar `class_cohorts.session_number`

`class_cohorts` é M:N **pure** entre classes e cohorts. Mesma `class` pode ser:
- "Aula 3 em Fundamentals T4"
- "Aula 5 em Fundamentals T5"

Adicionar `session_number` ali viola normalização — class compartilhada entre cohorts precisaria linhas com diferentes session_numbers ↔ mesma combinação class+cohort = chave única já existente; única opção seria duplicar relação ou desnormalizar.

### Schema proposto

```sql
CREATE TABLE cohort_sessions (
  id                     UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cohort_id              UUID NOT NULL REFERENCES cohorts(id) ON DELETE CASCADE,
  session_number         INT  NOT NULL CHECK (session_number > 0),
  class_id               UUID REFERENCES classes(id) ON DELETE SET NULL,
  planned_date           DATE,
  actual_zoom_meeting_id UUID REFERENCES zoom_meetings(id) ON DELETE SET NULL,
  status                 TEXT NOT NULL DEFAULT 'planned'
                              CHECK (status IN ('planned','live','done','cancelled')),
  notes                  TEXT,
  created_at             TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at             TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at             TIMESTAMPTZ,  -- soft delete pra audit
  CONSTRAINT cohort_sessions_unique UNIQUE (cohort_id, session_number)
);

CREATE INDEX idx_cohort_sessions_lookup     ON cohort_sessions(cohort_id, class_id)
                                            WHERE deleted_at IS NULL;
CREATE INDEX idx_cohort_sessions_planned    ON cohort_sessions(planned_date)
                                            WHERE status='planned' AND deleted_at IS NULL;
CREATE INDEX idx_cohort_sessions_zoom       ON cohort_sessions(actual_zoom_meeting_id)
                                            WHERE actual_zoom_meeting_id IS NOT NULL;

-- updated_at trigger
CREATE TRIGGER cohort_sessions_updated_at
  BEFORE UPDATE ON cohort_sessions
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### Snapshot histórico em `nps_class_links`

```sql
ALTER TABLE nps_class_links
  ADD COLUMN session_number_snapshot INT;

CREATE INDEX idx_nps_links_session_snap ON nps_class_links(session_number_snapshot)
  WHERE session_number_snapshot IS NOT NULL;
```

Dispatcher (`dispatch-class-nps`) preenche `session_number_snapshot` no momento do envio → label imutável mesmo se cohort remarcar depois.

### RLS

```sql
ALTER TABLE cohort_sessions ENABLE ROW LEVEL SECURITY;

CREATE POLICY cohort_sessions_read_admin ON cohort_sessions
  FOR SELECT TO authenticated
  USING (public.is_dashboard_admin());

CREATE POLICY cohort_sessions_write_admin ON cohort_sessions
  FOR ALL TO authenticated
  USING (public.is_dashboard_admin())
  WITH CHECK (public.is_dashboard_admin());
```

## Acceptance Criteria

```
GIVEN cohort_sessions populada para cohort X
WHEN nps_results_by_survey RPC resolve session_index
THEN lê de cohort_sessions.session_number (NÃO COUNT zoom_meetings)
  AND retorna mesmo valor independente de quantas zoom_meetings/fantasma o webhook criou

GIVEN dispatch NPS é enviado em data T
WHEN nps_class_links.session_number_snapshot é preenchido
THEN label histórica de respostas associadas a esse link permanece imutável
  AND se cohort remarcar Aula 3 → Aula 4 depois, labels antigas continuam "Aula 3"

GIVEN aluno responde NPS dia seguinte de aula
WHEN resposta é agrupada por dispatch_job_id
THEN "Aula NN" label vem do session_number_snapshot do link
  AND não é afetada por data da resposta ou meetings falsos

GIVEN cohort_sessions sem entrada pra (cohort, class)
WHEN RPC resolve session_index
THEN fallback ordem: snapshot > cohort_sessions > heurística legacy zoom_meetings
  AND retorna valor não-NULL mesmo pra dados antigos sem cohort_sessions populada

GIVEN admin edita session_number/planned_date via UI
WHEN cohort_sessions é atualizada
THEN audit trail preservado (updated_at + soft delete deleted_at)
  AND labels de envios futuros refletem mudança
  AND labels históricas (via snapshot) NÃO mudam

GIVEN cohort merge (relink class_cohorts)
WHEN cohort_sessions tem conflito de session_number
THEN ERROR explícito (não silencioso) — admin resolve manualmente
  AND zoom_meeting_id orphan é detectado (FK ON DELETE SET NULL)
```

## Scope

**IN:**
1. **Migration:**
   - Criar `cohort_sessions` table + indexes + RLS + trigger updated_at
   - `ALTER TABLE nps_class_links ADD COLUMN session_number_snapshot INT`
2. **Backfill script:**
   - Gerar CSV por cohort com proposta de `session_number` baseado em `zoom_meetings` filtrado (heurística atual)
   - Review manual ANTES apply (admin valida cada cohort)
   - Plus: detectar conflitos com cohort merges (memory: "Cohort merge relink classes")
3. **Refatorar `nps_results_by_survey` RPC:**
   - Resolve order: `lnk.session_number_snapshot` > `cs.session_number` > heurística legacy
   - Mantém fallback pra graceful degradation
4. **Atualizar 2 edge functions:**
   - `dispatch-class-nps`: preencher `session_number_snapshot` no insert do link
   - `nps-class-report-daily`: label "Aula NN" vem do snapshot
5. **Trigger sync `zoom_meeting_id`:**
   - Existente: `auto_set_zoom_meeting_class_id` (memory)
   - Estender: quando zoom_meeting.class_id é setado, FK pra cohort_session correta
6. **UI admin** (`/admin/cohort-sessions/` ou seção em cohort detail):
   - Listar sessions por cohort com session_number, planned_date, class, status
   - CRUD básico (add/edit/cancel session)
   - Drag-drop reorder (opcional v1 — pode ser input number direto)
7. **Tests SQL:**
   - `resolve_session_index()` function tests (snapshot > table > legacy)
   - RLS tests (authenticated admin vs non-admin)
   - Migration rollback test
8. **ADR-018:** `docs/architecture/ADR-018-cohort-sessions.md`

**OUT (próximas stories):**
- Bulk import via CSV — story separada
- PS RSVP integration com session_number — fora de scope (PS usa class_name, não session_number)
- Migrar session_index pra outras tabelas (envios admin etc) — story incremental

## Risks (revisados)

| Risco | Severidade | Mitigação |
|---|---|---|
| Backfill incorreto cria session_number errado pros 7 cohorts ativos | **High** | Dry-run report → review manual → confirmação por cohort. Sem auto-apply |
| Refactor RPC quebra UI dashboards existentes | High | Fallback dual `COALESCE(snapshot, table, legacy)`. Graceful degradation |
| Cohort merge conflito de session_number | Medium | Detectar no backfill + ERROR explícito (não silencioso) |
| nps_class_links históricos sem `session_number_snapshot` | Medium | RPC fallback resolve via cohort_sessions table → legacy heurística |
| zoom_meeting_id FK orphan se zoom deletado | Low | `ON DELETE SET NULL` |
| Trigger auto_set_zoom_meeting_class_id race condition | Medium | Idempotente (UPSERT em cohort_sessions.actual_zoom_meeting_id) |
| RLS bloqueia legacy callers | Low | Service-role bypass + GRANT SELECT pra authenticated com policy |
