## Descrição

Como **coordenador ou mentor**, quero fazer upload de slides e PDFs diretamente no painel de admin de aulas — para que os materiais fiquem disponíveis para os alunos logo após a aula — sem precisar usar links externos ou Google Drive manualmente.

**Contexto técnico:** `class_recordings` tem campos `chat_log TEXT` e `transcript_vtt TEXT` mas não tem suporte a arquivos. Precisamos criar um bucket no Supabase Storage (`class-materials`), uma tabela `class_materials(id, recording_id, file_name, file_url, file_type, uploaded_at)`, e uma interface de upload no admin de aulas.

---

## Acceptance Criteria

- [x] **AC1 — Upload de arquivo:** admin pode selecionar e fazer upload de PDF/PPTX/imagem (max 50 MB) para uma aula específica
- [x] **AC2 — Lista de materiais:** materiais uploaded ficam visíveis na aula com link para download/abertura
- [x] **AC3 — Tipos aceitos:** `.pdf`, `.pptx`, `.ppt`, `.key`, `.png`, `.jpg` (outros rejeitados com mensagem clara)
- [x] **AC4 — Remoção:** admin pode remover materiais já uploaded
- [x] **AC5 — Visível para alunos:** alunos logados no `/aulas` veem a lista de materiais com botão "Baixar" ou "Abrir"
- [x] **AC6 — Sem interferência:** upload de materiais é independente da gravação automática (ambos podem coexistir)

---

## Tasks

- [x] Criar bucket `class-materials` no Supabase Storage (público para leitura, auth para upload)
- [x] Criar migration `class_materials` — tabela com recording_id, file_name, file_url, file_type, file_size, uploaded_by, uploaded_at
- [x] Em `/aulas/index.html`: adicionar seção "Materiais" em cada aula com lista + botão "Adicionar Material" (só para admin)
- [x] Implementar `uploadMaterial(recordingId, file)` — usa Supabase Storage upload + INSERT em `class_materials`
- [x] Implementar `removeMaterial(materialId)` — DELETE em `class_materials` + Storage remove
- [x] Validar tipo de arquivo no frontend antes do upload
- [x] Exibir materiais para todos os usuários logados (não só admin)
