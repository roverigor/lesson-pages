# Story 4.2 — Edge Functions: submit-survey + dispatch-survey

**Epic:** EPIC-004  
**Status:** Ready  
**Type:** Backend / Edge Functions  
**Depende de:** Story 4.1

## O que fazer

Criar dois Edge Functions Supabase:
1. `submit-survey` — público, recebe resposta do aluno via token
2. `dispatch-survey` — protegido por admin JWT, gera tokens e envia WhatsApp

## Acceptance Criteria

### submit-survey (POST /functions/v1/submit-survey)
- [ ] Aceita `{ token: string, score: number, feedback?: string }`
- [ ] Valida token: busca em `survey_links` onde `token = $1 AND used_at IS NULL`
- [ ] Se token inválido ou já usado: retorna `{ error: 'token_invalid' }` com status 400
- [ ] Insere em `student_nps`: survey_id, student_id, cohort_id (da survey), score, feedback, survey_type, responded_at
- [ ] Atualiza `survey_links.used_at = now()`
- [ ] Usa SERVICE_ROLE_KEY (bypass RLS)
- [ ] Retorna `{ success: true }` com status 200
- [ ] CORS headers para `calendario.igorrover.com.br`

### dispatch-survey (POST /functions/v1/dispatch-survey)
- [ ] Requer Authorization Bearer JWT com role=admin
- [ ] Aceita `{ survey_id: string }`
- [ ] Busca survey (valida que existe e status='active')
- [ ] Busca alunos do cohort ou turma da survey (via `students` table com `cohort_id`)
- [ ] Para cada aluno: `INSERT INTO survey_links (survey_id, student_id) ON CONFLICT DO NOTHING`
- [ ] Para cada aluno com `phone` preenchido: envia WhatsApp via Evolution API
  - Mensagem: `"Olá {{nome}}! 👋\n\nSua opinião é muito importante para nós.\nResponda em 1 minuto: {{link}}\n\n_Academia Lendária_ 🚀"`
  - Link: `https://calendario.igorrover.com.br/avaliacao/responder?token={{token}}`
- [ ] Atualiza `surveys.dispatched_at = now()`
- [ ] Retorna `{ dispatched: N, skipped: M }` (N = enviados, M = sem phone)
- [ ] Alunos sem phone são incluídos em `survey_links` mas skipped no WhatsApp
