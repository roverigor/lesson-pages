## O que fazer

Bootstrap completo do repositório `cs-portal` (já criado em https://github.com/roverigor/cs-portal) com stack confirmada em ADR-015: vanilla JS + esbuild + SortableJS + ES Modules. Entregar Dockerfile production-ready, GitHub Actions deploy automático para VPS Contabo (porta 3081), estrutura de pastas conforme ADR, hash router, auth guard skeleton, e CLAUDE.md próprio para autonomia do time CS.

**Esta story destrava:** 15.5 (sidebar + tabs), 15.K (nginx routing), todas UIs subsequentes (15.2/15.3/15.4/15.6/15.7/15.D/15.F).

## Business Value

- **Autonomia real CS team (NFR-20):** repo separado + Claude Code próprio + CLAUDE.md scoped = CS opera sem tocar painel admin (cumpre promessa do épico)
- **Blast radius pequeno:** mudanças CS-portal não quebram lesson-pages; deploy/rollback isolados via container :3081 distinto
- **9 stories de UI desbloqueadas em cascata** (15.5/15.2/15.3/15.4/15.6/15.7/15.D/15.F + 15.K) = trabalho de UI destravado
- **Reuso máximo (Article IV):** vanilla JS + design tokens já existentes = zero invenção; bundle ~45 KB load-rápido
- **Quality embedded:** GitHub Actions CI/CD desde dia 1, smoke tests pós-deploy, branch protection main
- **Boundary governance:** CLAUDE.md repo CS impede CS team de tocar `supabase/migrations/*` por engano (CON-14 enforced)

### esbuild.config.js Skeleton

```javascript
import esbuild from 'esbuild';

const isWatch = process.argv.includes('--watch');

const config = {
  entryPoints: ['src/main.js'],
  bundle: true,
  outfile: 'public/dist/main.js',
  format: 'esm',
  target: 'es2022',
  minify: !isWatch,
  sourcemap: isWatch ? 'inline' : false,
  loader: { '.css': 'text' }
};

if (isWatch) {
  const ctx = await esbuild.context(config);
  await ctx.watch();
  console.log('Watching...');
} else {
  await esbuild.build(config);
  console.log('Built.');
}
```

### Hash Router Skeleton (src/main.js)

```javascript
import { authGuard } from './shared/auth-guard.js';

const routes = {
  '#/dashboard': () => import('./pages/dashboard.js'),
  '#/cohorts': () => import('./pages/cohorts.js'),
  '#/students': () => import('./pages/students.js'),
  '#/dispatch': () => import('./pages/dispatch.js'),
  '#/forms': () => import('./pages/forms-list.js'),
  '#/pending': () => import('./pages/pending.js'),
  '#/history': () => import('./pages/history.js'),
  '#/integrations': () => import('./pages/integrations.js')
};

async function navigate() {
  const hash = location.hash || '#/dashboard';
  const loader = routes[hash];
  if (!loader) return;

  const { default: render } = await loader();
  document.querySelector('#main-content').innerHTML = render();
}

window.addEventListener('hashchange', navigate);
window.addEventListener('load', async () => {
  if (!await authGuard()) return;
  navigate();
});
```

### Auth Guard Skeleton (src/shared/auth-guard.js)

```javascript
import { supabase } from './supabase-client.js';

export async function authGuard() {
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) {
    location.href = 'https://painel.igorrover.com.br/admin/login?redirect=/cs/';
    return false;
  }
  const role = user.user_metadata?.role;
  if (role !== 'cs' && role !== 'admin') {
    location.href = 'https://painel.igorrover.com.br/admin/login?error=unauthorized';
    return false;
  }
  return true;
}
```

```markdown
Stack: vanilla JS + esbuild + SortableJS + ES Modules. Sem framework reactivity.

## Boundaries CRÍTICAS

❌ NÃO modificar `supabase/migrations/*` (esse repo é lesson-pages)
❌ NÃO criar edge functions (também lesson-pages)

## Patterns

- ES Modules: `import`/`export` nativo
- Async/await > .then chains
- Sem virtual DOM — manipular DOM via Element APIs
- State management manual (event-driven simples) — sem Redux/Vuex
- Reuso de design tokens via @import na CSS
- Constants em UPPER_CASE (src/shared/constants.js)
- Funções exportadas nomeadas (não default exports — facilita refactor)

## Stories

Stories CS-portal específicas: 15.J (este bootstrap), 15.5 (sidebar), 15.6 (form builder), etc.
Stories backend: lesson-pages repo (15.A schema, 15.B/C/I edge functions).

## References

- Tech stack: docs/architecture/ADR-015 (em lesson-pages)
- Spec EPIC-015: docs/stories/EPIC-015-cs-area/spec/spec.md (em lesson-pages)
```

### Constraints Críticas

- ❌ **NÃO instalar framework reactivity** (Vue/React/Svelte) — viola ADR-015
- ❌ **NÃO copiar inteiro** lesson-pages css — usar @import (manter atualizações sincronizadas)
- ❌ **NÃO commitar** secrets em código — usar GitHub secrets + Supabase secrets
- ✅ Manter bundle < 50 KB (target ADR-015)
- ✅ ES Modules nativo (não bundlers que reescrevem para CommonJS)
- ✅ Estilo consistente com lesson-pages (dark premium)
