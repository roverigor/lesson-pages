## Descrição

Como **coordenador da Academia Lendária**, quero que o sistema capture automaticamente as gravações das reuniões Zoom e as salve no painel `/aulas` — sem que eu precise copiar links manualmente — para que 100% das aulas tenham gravação disponível.

**Contexto técnico:** O Zoom envia um webhook `recording.completed` quando a gravação fica disponível (~15-30 min após a reunião). A `zoom-webhook/index.ts` já existe e lida com outros eventos. Precisamos: (1) capturar `recording.completed`, (2) chamar a Zoom Cloud Recordings API para obter o link de download, (3) criar ou atualizar um registro em `class_recordings` associado ao cohort correto.

---

## Acceptance Criteria

- [x] **AC1 — Webhook capturado:** `zoom-webhook` processa `recording.completed` e não descarta o evento
- [x] **AC2 — Link de gravação salvo:** após o webhook, `class_recordings.video_url` contém o link de acesso à gravação Zoom Cloud (link de visualização, não download)
- [x] **AC3 — Cohort associado:** o registro em `class_recordings` é associado ao `cohort_id` correto usando o `host_email` da reunião (lookup em `zoom_meetings` ou `zoom_host_sessions`)
- [x] **AC4 — Deduplicação:** se o registro já existe (mesmo `meeting_id`), faz UPSERT (não duplica)
- [x] **AC5 — Duração salva:** `class_recordings.duration_minutes` preenchido a partir dos dados do webhook
- [x] **AC6 — Título da aula:** `class_recordings.title` preenchido com o `topic` da reunião
- [x] **AC7 — Estado na UI:** `/aulas` exibe a nova aula automaticamente após o webhook ser processado

## Risk Notes

- **Risk:** Gravação pode demorar > 30 min para processar no Zoom — o webhook vem depois
- **Mitigação:** Não há ação necessária; o webhook é assíncrono por design

- **Risk:** `host_email` não encontrado em `zoom_host_sessions`
- **Mitigação:** Salvar igualmente em `class_recordings` com `cohort_id = null`; admin pode associar manualmente depois
